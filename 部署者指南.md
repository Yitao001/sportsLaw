## 1. 项目简介

### 1.1 项目组成

SportsLax 是一个面向运动员和体育从业者的 AI 法律咨询智能体：

- **API 服务（api.py）**：处理法律咨询、报告生成、运动百科请求
- **RAG 引擎（rag/rag_engine.py）**：基于 ChromaDB 的向量知识库检索
- **运动百科（knowledge/sports_knowledge.py）**：14 个运动项目的规则与法律特点
- **模型工厂（model/factory.py）**：支持通义千问 / OpenAI / Claude / 硅基流动

### 1.2 核心功能

| 功能 | 端点 | 说明 |
|------|------|------|
| 智能法律对话 | POST /chat | RAG + LLM，支持流式输出 |
| 律师信息清单报告 | POST /report | 生成结构化法律报告 |
| 体育运动百科 | POST /sport | 查询运动法律特点 |
| 运动列表 | GET /sports/list | 获取收录运动列表 |
| 健康检查 | GET /health | 检查服务状态 |

### 1.3 技术栈

- **Web 框架**：FastAPI + Uvicorn
- **LLM 框架**：LangChain（支持多提供商）
- **向量数据库**：ChromaDB（本地持久化）
- **嵌入模型**：DashScope text-embedding-v4 / OpenAI Embeddings
- **容器化**：Docker + Docker Compose

## 3. 本地部署

### 3.1 克隆项目

```bash
git clone https://github.com/Yitao001/sportslax.git
cd sportslax
```

### 3.2 创建虚拟环境

```bash
python -m venv venv
# Windows
venv\Scripts\activate
# Linux / macOS
source venv/bin/activate
```

### 3.3 安装依赖

```bash
pip install -r requirements.txt
```

### 3.4 配置环境变量

```bash
cp .env.example .env
# 编辑 .env，填入 API Key
```

### 3.5 启动服务

```bash
python api.py
# 或
uvicorn api:app --reload --port 8000
```

### 3.6 访问

- Swagger UI：`http://localhost:8000/docs`
- 健康检查：`http://localhost:8000/health`

## 4. Docker 部署

### 4.1 构建并启动

```bash
docker compose up -d
```



### 5.1 前置要求

- 阿里云 ECS 或任意云服务器（2核2G以上）
- Python 3.10+ / Docker（可选）

确保开放以下端口：

- **8000**：API 服务（HTTP）

## 6. 配置说明

### 6.1 .env 配置项

```env
# 模型提供商
MODEL_PROVIDER=tongyi          # 可选：tongyi / openai / siliconflow / anthropic
CHAT_MODEL_NAME=qwen-max       # 对话模型名称
EMBEDDING_MODEL_NAME=text-embedding-v4  # 嵌入模型名称

# 通义千问（阿里云）
DASHSCOPE_API_KEY=your_key_here

# OpenAI 兼容接口
# OPENAI_API_KEY=sk-xxx
# OPENAI_BASE_URL=https://api.openai.com/v1

# 安全配置
API_KEY=                        # 空为开发模式，填入启用认证
CORS_ORIGINS=*                 # 允许的跨域来源
RATE_LIMIT_PER_MINUTE=60       # 速率限制
```

### 6.2 RAG 知识库

向量数据库持久化在 `data/chroma_db/`，首次启动时自动构建内置知识库。

自定义文档追加：

```python
from langchain_core.documents import Document
from rag.rag_engine import add_custom_documents

docs = [
    Document(
        page_content="自定义法律文本内容...",
        metadata={"source": "来源名称", "category": "分类"}
    )
]
add_custom_documents(docs)
```

## 8. 故障排查

### 问题 1：服务启动失败

- 检查端口 8000 是否被占用：`netstat -tlnp | grep 8000`
- 检查 `.env` 配置是否正确
- 检查 `requirements.txt` 依赖是否完整安装

### 问题 2：LLM 调用报错

- 确认 API Key 正确且有效
- 确认网络可以访问模型提供商的 API
- 检查 `.env` 中 `MODEL_PROVIDER` 配置是否匹配实际 API

### 问题 3：向量知识库检索为空

- 确认 `data/chroma_db/` 目录存在且有数据
- 首次启动会自动构建内置知识库，等待启动完成后再测试

### 问题 4：Docker 部署内存不足

- 增加容器内存限制，或使用 `docker compose up --memory` 限制
- ChromaDB 在 Docker 中运行需要至少 512MB 内存
